# This file contains the definitions used to create the options used by the code
# formatter (and autocomplete, and a few other things). The file is fed into
# formatter_options.pl, which generates the options by processing inlined Perl
# in the template files. Here are the templates currently in use:
#
# DefaultCodeFormatterConstants2.template.java -> ../src/descent/core/formatter/DefaultCodeFormatterConstants2.java
# DefaultCodeFormatterOptions2.template.java -> ../src/descent/internal/formatter/DefaultCodeFormatterOptions2.java
#
# The syntax for inlined perl in the template files is fairly strict - to
# evaluate a block of Perl for each option encountered in this file, start a
# block comment with "/* EVAL-FOR-EACH", which must be on the same line.
# Any lines until a line containing "*/" that start with a "* " (note the space)
# will be evaled as perl. This implies that the line with the "*/" on it is not
# evaled. It's a bit hackish, but it works, right?
#
# The syntax of this file is also pretty strict. If a line starts with a #, that
# line is ignored (or if the line is blank), otherwise it is processed. Lines
# without an "=" are treated as the beginnings of definitions, lines with an "="
# are treated as parts of the definition, with name=value pairings. The first
# line of the definition (the onw ithout the "=" is considered the option name,
# accessed with the key "optName". There's also "constName", which is the
# uppercase version of "optName", with "FORMAT_" appended.
#
# In the inlined perl, DST is the destination file handle, and $_ is a hash
# reference to the settings for that option. So, $$_{'optName') is the option
# name and if the option definition had a "type" property, $$_{'type'} is used
# to access it.
#
# Hopefully that isn't too hard to understand (you know, I porbably spent longer
# typing this up than it would have taken me to fix the damn script).
#
#
#
# For options, these are the meanings of the various hash keys used so far:
#     optName       The name of the option used in an instance of DefaultCodeFormatterOptions
#     constName     The name of the constant string for Java map keys used in DefaultCodeFormatterConstants
#     type          The type of the option in DefaultCodeFormatterOptions
#     default       The default value (we may want to have multiple profiles...?)
#
# The hope is this can be expanded for use in UI, etc., etc.
